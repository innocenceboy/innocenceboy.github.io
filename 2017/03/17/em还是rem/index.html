em还是rem

发表于 2017-03-17 | 分类于 CSS

首先介绍下这两个单位，em和rem都是相对单位，由浏览器转换为像素值，具体取决于您的设计中的字体大小设置。看下面的例子：

首先你要知道的是每个浏览器有一个默认的font-size，比如Chrome浏览器里默认的font-size为16px，所以这里计算html元素的font-size时1rem=16px，使得html元素的font-size值为64px。

接着当计算body的font-size时，此时的1rem=64px，所以body元素的font-size值为5 * 64 = 320px。

当计算div.wrapper的时候，此时的1rem=64px，html元素作为页面的根元素，rem以它的font-size为基准。所以div.wrapper的font-size为192px。

同理，对于最内层的div来说，1rem=64px，所以font-size为192px，padding-left为128px。

还是同样的例子，只不过单位换成了em。

同样的，这里计算html元素的font-size时1em=16px，使得html元素的font-size值为64px。

接着当计算body的font-size时，根据继承，此时的1em=64px，所以body元素的font-size值为320px。

计算div.wrapper时，根据继承，此时1em=320px，所以div.wrapper的font-size值为960px。

计算最内层的div时，根据继承，此时1em=960px，所以最内层div的font-size值为2880px，注意，由于它的font-size值被显示设置了，所以此时1em=2880px，padding-left的值为5760px。

总结如下：

  1. 每个浏览器有一个默认的font-size，例如Chrome浏览器的为16px。
  2. html元素作为页面的根元素，不管使用em还是rem，都是以浏览器默认的font-size为基准。如果html元素显式的设置了font-size，它的所有的后代元素中，使用rem的都将以它的font-size为基准，否则以浏览器默认的font-size为基准。
  3. font-size具有继承性，如果某个元素某个属性使用em且没有显式设置font-size时，那么会把继承而来的font-size作为基准，否则会以自己的font-size作为基准。
  4. 如果某个元素显式设置了font-size，且单位为em，那么会使用继承值计算出自己的font-size。

理解viewport

发表于 2016-12-27 | 分类于 技术栈

在查阅ppk大神写的几篇关于viewport的文章后，我是这么理解移动端下viewport这么个“磨人的小妖精”的。先上一张图：

结论：visual viewport的大小表示能够“看到”多大的layout viewport，如果它们大小相等，那么透过visual viewport就能“看到”完整的layout viewport；如果visual viewport比layout viewport小，那么只能透过visual viewport“看到”一部分的layout viewport；如果visual viewport比layout viewport大，除了能透过visual viewport“看到”完整的layout viewport，还会包括之外的范围。
拿iphone5为例，官方分辨率参数为1136*640，这是指物理像素，设备像素比为2，所以一个css像素由4个物理像素表示，css屏幕大小为568*320，通过下面的案例理解一下：

根据ppk大神的讲解，layout viewport的宽高可由document.documentElement.clientWidth和document.documentElement.clientHeight得出，在iphone5下是
1739 * 980，这也是大多数机型采用的设置；visual viewport的宽高可由window.innerWidth和window.innerHeight得出，这里是1739 * 980；屏幕大小可由screen.width和screen.height得出，这里是568 * 320。
所有布局都是相对于layout viewport的，所以上面宽度为490的方块应该占据一半的layout viewport，由于visual viewport的大小和layout viewport大小相等，所以从visual viewport这个“小窗口”能看到全部的layout viewport。
添加如下标签：
<meta name="viewport" content="initial-scale=2.0">
发现layout viewport的宽度变为160，visual viewport的宽度为160，屏幕宽度仍为320，所以initial-scale=2是屏幕和visual viewport之间的比例，用下面的公式表示：
initial-scale = ideal viewport / visual viewport
initial-scale = screen.width / window.innerWidth
没有设置width的时候，layout viewport和visual viewport的大小一样，当有width的时候呢？
添加如下标签：
<meta name="viewport" content="width=980, initial-scale=2.0">
上面标签中的width=980表示想设置layout viewport的宽度为980，此时visual viewport的宽度为160，屏幕宽度仍为320。
当width和initial-scale同时出现的时候，取计算之后的最大值。比如width为120，屏幕宽度为320，那么计算之后的结果为160，160大于120，此时layout viewport的宽度应为160；比如width为170，屏幕宽度为320，那么计算之后的结果为160，160小于170，此时layout viewport的宽度应为170。
在没有设置上面的meta标签的时候，浏览器会自动计算一个默认的initial-scale值，从而使得浏览器不会出现滚动条。比如上面的例子中，visual viewport和layout viewport大小都为980，initial-scale的值为0.33(320/980)左右。
ppk大神认为有三个viewport存在，分别是ideal viewport、visual viewport和layout viewport，其实屏幕就是ideal viewport，它们之间的关系可以总结为：visual viewport表示可以显示多大的layout viewport，之后再缩放成为屏幕大小。
那么如何让三个viewport大小相等，通过下面的标签设置：
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
上面标签的意思如下：让layout viewport的宽度等于屏幕也就是ideal viewport的宽度，通过initial-scale控制visual viewport的宽度等于ideal viewport的宽度，从而让它们三个大小相等。
遇到有趣的问题：我设置了如下标签：
<meta name="viewport" content="initial-scale=0.1">
理想中visual viewport值为3200，且没有设置width，所以layout viewport值取visual viewport的值，这里取3200，但是实际上两者都为1280，说明有一个上限。

总结：

  1. 没有设置<meta>标签的时候，有一个初始的initial-scale的值。
  2. 没有设置<meta>标签的时候或者设置了<meta>标签但只设置了initial-scale的值，layout viewport和visual viewport相等。
  3. initial-scale = ideal viewport / visual viewport
  4. initial-scale = screen.width / window.innerWidth
  5. 设置了<meta>标签并且设置了initial-scale的值和width的值，那么经过计算后layout viewport的值取visual viewport和width之间的最大值。
